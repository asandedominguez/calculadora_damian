Paso 1

P1: Se hace clone porque así se descarga también el historial y la conexión con GitHub. Copiar archivos no vincula el proyecto al repositorio.
P2: PyCharm muestra colores en los archivos, iconos de Git y la rama actual en la barra inferior.

Paso 2

P1: El push falla porque el remoto tiene cambios nuevos en la misma línea que tú modificaste.
P2: Git no elige automáticamente porque no sabe qué versión es correcta; necesita decisión humana.

Paso 3

P1: Un conflicto es cuando dos cambios afectan a la misma parte del código y Git no puede fusionarlos solo.
P2: Tras resolver: guardar → commit → push al repositorio remoto.

Paso 4

P1: El mejor workflow para evitar conflictos es trabajar con ramas de funciones (feature branches) y hacer pull requests con revisiones antes de mezclar los cambios.
